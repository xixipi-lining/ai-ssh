# 🚀 AI-SSH: 无缝的 AI 命令行伴生助手

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](https://opensource.org/licenses/MIT)

在终端输入自然语言，按下快捷键，直接原地替换为精准的 Shell 命令。
支持本地直调与 SSH 远程无缝映射。**零安装、零配置、凭证绝对安全。**

---

## 🤔 为什么写这个工具？

在服务器运维和本地开发中，记忆复杂的 Shell 命令（如 `tar` 或 `awk` 参数）往往是一大痛点。虽然当前市面上有许多优秀的 AI 解决方案，但在实际的**终端场景**中操作往往不够顺畅：

- 📉 **方案一：依赖网页端对话（如 ChatGPT 网页版）**
  你需要切换浏览器，手动输入环境背景（如系统类型、所在目录等），复制生成的代码，再切回终端粘贴。工作流经常被打断。

- 🔑 **方案二：使用独立命令行 AI 工具（如 `shell-gpt` 等）**
  在本地使用体验较好，但一旦通过 SSH 连接到远程服务器，你要么在每台服务器上重新安装环境依赖，要么将个人的 API Key 配置在不受信任的远程机器上。这既繁琐，又带来安全隐患。

- 💸 **方案三：使用内置 AI 的专用终端软件（如 Warp）**
  体验优秀，但通常属于增值订阅服务（需支付额外费用），且将你绑定在特定的终端生态中。

- 🛑 **方案四：切换到全新的 AI 驱动新型终端（如 Wave）**
  功能强大，但也引入了全新的使用逻辑。需要重新适应工具生态和操作习惯，对于适应了原生终端的用户来说迁移成本较高。

**AI-SSH 的设计理念：不造大模型的轮子，专注解决连接问题。**
它基于现有 AI 工具的功能接口，通过底层的「按键劫持」与「SSH 动态隧道」，把你**本地终端已有的 AI 算力和配置**，安全地映射到任意远程终端光标处。**用完即走，不留痕迹。**

## 🌟 核心特性与设计理念

为了提供更好的使用体验，AI-SSH 遵循以下设计原则：

1. ⌨️ **原位替换 (`Ctrl+G`)**：输入自然语言，按下快捷键会自动替换为待执行命令。配合 **带剪贴板求助 (`Ctrl+B`)** ，还能自动读取系统剪贴板（如报错日志）并发送给 AI，快速定位问题。**在命令行无输入时直接按快捷键，更可呼出本地原生输入对话框，彻底告别服务器高网络延迟下的打字卡顿。**
2. 🛡️ **Zero-Footprint (远程零侵入)**：不向远程服务器写入任何永久配置、脚本或身份凭证。核心的 Bridge 进程在后台静默运行，随着 SSH 连接断开时自动关闭清理。
3. 🧠 **Context-Aware (上下文感知)**：AI-SSH 会自动采集当前环境的变量（如操作系统、当前所在目录 `$PWD` 等）拼装给 AI。这使得输出结果能够直接适配当前环境。
4. ⛓️ **Graceful Degradation (自动降级)**：为了应对多样的服务端 `sshd_config` 限制，程序在后台会同时建立 Socket 和 TCP 双通道隧道。远程 Hook 会进行连通性检测并自动降级。

## 🔌 支持模式：API 与 Client 双引擎

针对不同的 AI 服务使用场景，AI-SSH 支持以下两种工作模式：

### 模式一：API 直连模式

依赖系统自带 Python 的内置库发起请求，无需额外安装 CLI 工具。适用于持有 OpenAI、DeepSeek、Google AI Studio 等兼容平台 API Key 的用户。

- 你只需在配置中设定 `AI_TOOL="openai"` 和对应的 `OPENAI_API_KEY`。

### 模式二：客户端代理模式

适用于依赖特定客户端工具或基于本地环境 OAuth 授权登录的用户群体。

- 比如你开通了 Google 官方订阅或 Claude 服务，可以在宿主机上安装好对应的 `gemini-cli` 或 `claude-code`。配置中指定 `AI_TOOL="gemini"` 后，即可复用本地的认证与额度体系，无需再次配置密钥。类似客户端也支持 `kimi` 等。

## 📦 快速安装

```bash
git clone https://github.com/xixipi-lining/aissh.git
cd aissh
bash install.sh
```

> **安装脚本做了什么？**
>
> 1. 将组件代码存放在 `~/.ai-ssh/` 目录下。
> 2. 自动检测并在 `~/.zshrc` 或 `~/.bashrc` 内注入配置快捷键所需代码。
> 3. 将 `aissh` 命令软连接到 `$PATH` 环境变量中。

## ⚙️ 配置方法

**前置依赖**：本项目专注“连接映射”。你的**本地宿主机**只需要装有 `Python 3` 和 `curl`。

初始化后，请编辑 `~/.ai-ssh/config`，告诉 AI-SSH 你想使用什么引擎：

```ini
# --- 模式一：Client 客户端代理 ---
# AI_TOOL="gemini"    # 填入工具名即可，支持: gemini, kimi, claude

# --- 模式二：API 直连模式 ---
AI_TOOL="openai"
OPENAI_API_BASE="https://api.openai.com/v1"
OPENAI_API_KEY="sk-your-api-key"
OPENAI_MODEL="gpt-4o"
```

## 🖥️ 使用指南

### 场景 A：本地开发 (Local)

打开终端，像平时一样打字：

```bash
$ 找出当前目录下超过 500M 的文件，打包后删除它们   # 输入完不要回车，直接按 Ctrl+G
$ find . -type f -size +500M -exec tar -czvf large_files.tar.gz {} + -delete  # ⬅️ 瞬间替换
```

_如果你刚复制了一段复杂的报错信息栈，你可以输入“看看是什么报错引起的”，然后按 **`Ctrl+B`**。_

### 场景 B：远程运维 (SSH)

使用 `aissh` 命令平替原生的 `ssh` （支持所有原生的 ssh 连接参数组合）：

```bash
$ aissh proxy-user@192.168.1.100 -p 2222
```

登录成功后，远程服务器上的体验将**与本地保持一致**。

```bash
remote-user@host:~$ 查看 nginx 这个进程当前的活动网络端口    # 方式一：输入完按下 Ctrl+G (原位替换)
remote-user@host:~$ sudo netstat -tulnp | grep nginx
remote-user@host:~$ exit                       # 退出连接时，本地伴生的微服务跟随自动清理
```

**【🌟 远程高延迟优化：零延迟本地弹框模式】**

如果连接的是海外或高延迟服务器，在终端里直接打字回显都很卡时，你可以：

👉 **不输入任何字符，直接按下 `Ctrl+G`（或 `Ctrl+B`）**

由于核心 Bridge 微服务运行在你的宿主机上，AI-SSH 会瞬间在你的**本地实体机**弹出一个无依赖的系统原生对话框。你在本地流畅地输入你的自然语言意图并按下发送（macOS 环境下为 `Enter` 发送，`Option+Enter` 换行），指令会毫秒级返回并填充在远程终端光标处，完全消灭网络卡顿感！

## 🏗️ 架构概览与数据流

融合了原本 `DESIGN.md` 中的设计，AI-SSH 的架构做到了高度的松耦合：

```text
[ 本地物理机 (Local Machine) ]                     [ 远程服务器 (Remote Server) ]

 ╭─────────────────────────────────╮
 │ 【配置与底座】 ~/.ai-ssh/config   │<- (模式1: OpenAI API / 模式2: Client CLI)
 ╰─────────────────────────────────╯
                 ▲
  [本地直连]      │[跑远程时: aissh 伴生拉起]
  (绕过网络)      │                  │
 ╭───────────────┴─╮      ╭──────────┴─────────────╮
 │ 1. Local Hook   │      │ 2. Ephemeral Bridge    │ (解析 JSON，发起底座请求)
 │ (本地环境劫持)    │      │ (随 aissh 启动的微服务) │
 ╰────────┬────────╯      ╰──────────┬─────────────╯
          │                          ▲
          │                 ┌────────┴─────────[-R 加密双隧道] ───────┐
          │                 │ 线路A: 随机 Socket (/tmp/aissh-xxx)   │
          │                 │ 线路B: 随机 TCP 端口 (127.0.0.1:PORT)  │
          │                 └────────▲──────────────────────────────┘
          │                          │ (智能连通性检测: 优先A，降级B)
          └─(收集 OS/PWD/ENV)        │
                     双热键触发: ╭────┴───────────────╮
[Ctrl+G] 仅当前行自然语言          │ 3. Remote Hook     │ (纯净内存级远程终端劫持)
[Ctrl+B] 当前行 + 剪贴板内容       ╰────────────────────╯
```

数据流转述：
当远程快捷键被激活后，内存中的 `hook.sh` 打包 `$PWD`、$OS 环境变量等作为上游 Context，沿着已架设好的 SSH 隧道，发回本地物理机的 `bridge.py` 端口。本地物理机调度你的 API / CLI 进行大模型计算。结束后，返回的纯净命令文本原路折返，打印在你远程会话的当前光标下。
